# Documentation / 文档入口

Open Test Sandbox keeps the core generic and local-first. Start with the
shortest path for your role.

Open Test Sandbox 保持核心通用、本地优先。你可以按自己的角色从下面的最短路径进入。

## New Users / 新用户

- [Quick Start](quickstart.md): install dependencies, create a SQL Store,
  register and verify an environment, run the workbench, and verify the
  checkout. 安装依赖、创建 SQL Store、登记并验收环境、启动工作台并验证仓库。
- [Share Kit](share-kit.md): project tagline, short descriptions, demo script,
  and announcement snippets. 项目 tagline、短介绍、demo 脚本和传播文案。
- [API Case Format](api-case-format.md): write one runnable HTTP case and
  understand the Evidence files it creates. 编写一个可运行的 HTTP 用例，并理解生成的 Evidence。
- [Store Backends](store-backends.md): configure PostgreSQL or MySQL Stores for
  personal and team workflows, with SQLite kept only for compatibility.
  配置个人与团队工作流使用的 PostgreSQL 或 MySQL Store，并了解 SQLite 兼容路径。
- [Backend Capabilities](backend-capabilities.md): complete overview of Store,
  import bundle publishing, discovery, execution, reports, Evidence, Control plane
  APIs, and release guardrails. 完整了解 Store、配置发布、目标发现、执行、报告、证据、控制平面 API 和发布守卫。

## Import Bundle Authors / 配置包作者

- [Import Bundle Format](import bundle-format.md): manifest fields, split assets, audit,
  install, pack, import, and verify. 配置清单字段、拆分资产、审计、安装、打包、导入和验证。
- [Import Bundle Authoring Guide](import bundle-authoring.md): practical workflow for
  keeping team-owned test assets outside the core repository. 如何把团队测试资产维护在核心仓库之外。

## Agent and CI Integrators / Agent 与 CI 接入方

- [CLI and API Contracts](cli-api-contracts.md): discovery, single-interface
  reports, workflow reports, asynchronous batch reports, and failed-case
  Evidence lookup. 目标发现、单接口报告、工作流报告、异步批量报告和失败证据查询。
- [Release Checklist](release-checklist.md): local and CI gates before sharing
  a public tag. 公开 tag 前的本地和 CI 门禁。

## Project Overview / 项目总览

- [Roadmap](roadmap.md): public development themes and contribution-friendly
  milestones. 公开迭代主题和适合贡献的里程碑。
- [Core Capabilities, Skills, and Goals](core-capabilities-skills-goals.html):
  visual overview of the current workbench capabilities and direction.
  当前工作台能力和方向的可视化总览。
